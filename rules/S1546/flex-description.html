Description

Every class definition should be commented to explain its goal and how it works. This comment must be located before [or after] the class definition.

Noncompliant Code Example

CLASS my_class DEFINITION.
  ...
ENDCLASS

Compliant Solution

* here is
* my comment
CLASS my_class DEFINITION.
  ...
ENDCLASS
